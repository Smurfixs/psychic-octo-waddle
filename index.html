Логическая структура базы данных

База данных системы контроля качества металлопродукции включает следующие основные сущности:

  • Пользователи (users) — учётные записи операторов, инженеров и администраторов системы
  • Рулоны (rolls) — информация о произведённых рулонах металлопродукции
  • Дефекты (defects) — зарегистрированные дефекты на рулонах
  • События (events) — журнал всех действий в системе

Физическая структура базы данных

Данные хранятся в локальном хранилище браузера (LocalStorage) в формате JSON. Каждая таблица представлена как массив объектов с уникальными идентификаторами.

users (Пользователи)

idINTEGER PK
usernameVARCHAR(50)
passwordVARCHAR(255)
fullNameVARCHAR(100)
roleENUM
emailVARCHAR(100)
createdAtDATETIME

rolls (Рулоны)

idINTEGER PK
numberVARCHAR(20)
steelGradeVARCHAR(20)
thicknessDECIMAL(5,2)
widthINTEGER
weightDECIMAL(10,2)
productionDateDATE

defects (Дефекты)

idINTEGER PK
rollIdINTEGER FK
typeVARCHAR(50)
criticalityENUM
statusENUM
positionDECIMAL(10,2)
descriptionTEXT
createdAtDATETIME
createdByINTEGER FK

events (События)

idINTEGER PK
typeENUM
descriptionTEXT

userNameVARCHAR(100)
createdAtDATETIME

Обоснование проектирования БД

Структура базы данных разработана с учётом следующих принципов:

  • Нормализация: данные приведены к 3НФ для исключения избыточности
  • Целостность: внешние ключи обеспечивают связность данных
  • Расширяемость: структура позволяет добавлять новые типы дефектов и атрибуты
  • Производительность: индексы на часто используемых полях (id, rollId, createdAt)
